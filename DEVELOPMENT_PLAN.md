# PSiteDL 开发计划

## 任务目标
优化 PSiteDL 项目的 5 个方向，建立 TDD 模式，完善文档

## 改进方向

### 1. 错误处理
- [ ] 网络异常重试机制 (超时、DNS 失败、连接重置)
- [ ] 页面结构变化容错 (正则匹配失败时的降级策略)
- [ ] Cookie 导出失败的优雅处理
- [ ] yt-dlp 未安装的自动安装/提示

### 2. 进度显示
- [ ] 下载进度百分比显示
- [ ] 下载速度实时显示
- [ ] 预计剩余时间
- [ ] 多文件下载进度条

### 3. 并发下载
- [ ] URL 文件批量并行下载
- [ ] 可配置并发数 (默认 3)
- [ ] 失败重试队列
- [ ] 下载结果汇总报告

### 4. 配置管理
- [ ] JSON/YAML 配置文件 (~/.psitedl/config.json)
- [ ] 配置项：输出目录、浏览器、profile、并发数、重试次数
- [ ] CLI 参数覆盖配置文件
- [ ] 配置初始化向导

### 5. 日志系统
- [ ] 结构化日志 (JSON 格式可选)
- [ ] 日志级别：DEBUG/INFO/WARNING/ERROR
- [ ] 日志文件轮转 (按大小/日期)
- [ ] 关键操作审计日志

## TDD 测试计划

### 测试框架
- pytest >= 7.0
- pytest-asyncio (异步测试)
- pytest-cov (覆盖率报告)
- responses (HTTP 模拟)

### 测试覆盖目标
- 核心功能覆盖率 >= 80%
- 错误处理分支 100% 覆盖
- 所有公共 API 都有测试

### 测试文件结构
```
tests/
├── __init__.py
├── conftest.py              # 共享 fixtures
├── test_site_cli.py         # CLI 功能测试
├── test_probe.py            # 视频探测测试
├── test_cookies.py          # Cookie 导出测试
├── test_config.py           # 配置管理测试
├── test_downloader.py       # 下载器测试
├── test_logging.py          # 日志系统测试
└── fixtures/
    ├── sample_page.html     # 测试页面
    └── config_sample.json   # 配置样例
```

## 开发流程

1. **Phase 1**: 搭建测试框架 (1-2 天)
   - 创建 tests/ 目录结构
   - 配置 pytest
   - 编写首批测试用例

2. **Phase 2**: 实现配置管理和日志系统 (2-3 天)
   - 先写测试
   - 实现功能
   - 通过测试

3. **Phase 3**: 实现错误处理和进度显示 (2-3 天)
   - 先写测试
   - 实现功能
   - 通过测试

4. **Phase 4**: 实现并发下载 (2-3 天)
   - 先写测试
   - 实现功能
   - 通过测试

5. **Phase 5**: 文档完善 (1 天)
   - 更新所有文档
   - 添加示例代码
   - 编写 FAQ
