## 📋 版本目标

基于 v0.4.0 的稳定基础，v0.5.0 聚焦于 **用户体验增强** 和 **功能完善**。

### 核心目标
1. **GUI 界面增强** - 提升可视化效果和任务管理能力
2. **断点续传增强** - 改进下载状态管理和恢复机制

4. **国际化支持** - 为多语言用户做准备

## 🎯 功能需求

### Feature 1: GUI 进度可视化增强

#### 用户故事
作为用户，我希望在下载过程中看到更直观的进度可视化，包括实时速度图表、剩余时间预测和任务队列状态。

#### 功能规格
- **实时速度图表**: 使用 matplotlib 或 pyqtgraph 显示下载速度曲线
- **剩余时间预测**: 基于当前速度和剩余大小智能预测 ETA
- **任务队列视图**: 显示等待中、下载中、已完成、失败的任务列表
- **暂停/恢复控制**: 支持单个任务和全局任务的暂停/恢复

#### 验收标准 (AC)
```gherkin
场景 1: 实时速度图表显示
  当 下载开始时
  那么 GUI 显示实时速度曲线图
  并且 图表每秒更新一次
  并且 显示当前速度、平均速度、峰值速度

场景 2: 剩余时间预测
  当 下载进行超过 10 秒
  那么 显示预计剩余时间 (ETA)
  并且 ETA 根据当前速度动态调整
  并且 预测误差在 ±20% 以内

场景 3: 任务队列管理
  当 批量下载时
  那么 显示所有任务的状态（等待/下载中/完成/失败）
  并且 支持点击暂停单个任务
  并且 支持点击恢复暂停的任务

场景 4: 全局控制
  当 点击"全部暂停"按钮
  那么 所有正在下载的任务暂停
  并且 已暂停的任务保持暂停状态
  当 点击"全部恢复"按钮
  那么 所有暂停的任务恢复下载
```

#### 技术约束
- 使用 PyQtGraph 或 Matplotlib 进行图表渲染
- 图表更新频率不超过 1Hz（避免 UI 卡顿）
- 与现有 downloader.py 的 BandwidthLimiter 集成

### Feature 2: 断点续传增强

#### 用户故事
作为用户，我希望在下载中断后能够无缝恢复，并且系统能自动保存和恢复下载状态。

#### 功能规格
- **状态文件持久化**: 每 30 秒自动保存下载状态到 `.psitedl-state` 文件
- **崩溃恢复**: 程序崩溃后重启时自动检测并恢复未完成的下载
- **状态验证**: 恢复时验证已下载片段的完整性（哈希校验）
- **手动恢复**: 支持手动指定状态文件恢复下载

#### 验收标准 (AC)
```gherkin
场景 1: 自动状态保存
  当 下载进行时
  那么 每 30 秒自动保存状态文件
  并且 状态文件包含：已下载片段、总大小、URL、配置

场景 2: 崩溃后自动恢复
  当 程序崩溃后重启
  那么 检测到未完成的状态文件
  并且 提示用户是否恢复下载
  当 用户确认恢复
  那么 从上次中断的位置继续下载
  并且 验证已下载片段的完整性

场景 3: 手动恢复
  当 用户指定状态文件路径
  那么 加载状态文件并恢复下载
  并且 显示恢复进度

场景 4: 状态文件清理
  当 下载完成
  那么 自动删除状态文件
  当 下载失败且超过 7 天
  那么 自动清理过期状态文件
```

#### 技术约束
- 状态文件使用 JSON 格式存储
- 哈希校验使用 MD5 或 SHA-256
- 状态文件存储在输出目录的 `.psitedl-state` 子目录

### Feature 3: 视频探测算法优化

#### 用户故事
作为用户，我希望工具能自动探测更多网站的视频资源，包括新的流媒体格式和加密视频。

#### 功能规格
- **新增格式支持**: HLS (m3u8) v8、DASH (mpd) 动态自适应流
- **JavaScript 渲染支持**: 使用 Playwright 处理动态加载的视频
- **加密视频检测**: 检测 DRM 保护的视频并提示用户
- **批量探测**: 支持批量 URL 的视频资源探测

#### 验收标准 (AC)
```gherkin
场景 1: HLS v8 格式支持
  当 页面包含 HLS v8 格式的 m3u8 链接
  那么 工具能正确提取并解析
  并且 支持多码率自适应选择

场景 2: DASH 动态流支持
  当 页面包含 DASH 格式的 mpd 链接
  那么 工具能正确提取并解析
  并且 显示可用的视频/音频轨道

场景 3: JavaScript 渲染
  当 视频链接通过 JavaScript 动态加载
  那么 使用 Playwright 等待并捕获
  并且 支持配置等待超时时间

场景 4: DRM 检测
  当 页面包含 DRM 保护的视频
  那么 提示用户"此视频受 DRM 保护，无法下载"
  并且 不提供下载选项
```

#### 技术约束
- 使用 Playwright 进行浏览器自动化
- 保持与现有 cookie 导出功能的兼容
- 探测超时时间可配置（默认 30 秒）

---

### Feature 4: 国际化 (i18n) 支持

#### 用户故事
作为非英语用户，我希望使用自己的语言与工具交互。

#### 功能规格
- **多语言框架**: 使用 gettext 或类似方案
- **初始语言**: 简体中文 (zh_CN)、英语 (en_US)
- **翻译范围**: CLI 输出、GUI 界面、错误消息、日志消息
- **语言切换**: 支持通过配置或环境变量切换语言

#### 验收标准 (AC)
```gherkin
场景 1: 语言配置
  当 用户设置 LANG=zh_CN
  那么 CLI 输出显示为简体中文
  当 用户设置 LANG=en_US
  那么 CLI 输出显示为英语

场景 2: GUI 语言切换
  当 用户在 GUI 中选择语言
  那么 界面立即切换为所选语言
  并且 语言设置持久化

场景 3: 错误消息翻译
  当 发生错误时
  那么 错误消息以用户选择的语言显示
  并且 错误代码保持一致

场景 4: 日志语言
  当 用户配置日志语言
  那么 日志消息以指定语言记录
  默认 日志使用英语（便于调试）
```

#### 技术约束
- 使用 Python gettext 模块
- 翻译文件存储在 `locales/` 目录
- 保持向后兼容（默认英语）
